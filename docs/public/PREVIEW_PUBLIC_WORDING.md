# Codiquiz Preview Public Wording

## Purpose

Use consistent wording when describing the live preview publicly, in a README, portfolio page, CV bullet, LinkedIn message, or recruiter conversation.

## Safe description

```text
Codiquiz is an early-alpha AI-powered coding quiz platform. The live preview demonstrates the public site, admin workflows, taxonomy system, Blueprint planning, AI question generation pipeline, quality checks, background workers, and deployment foundation.
```

## Short portfolio wording

```text
Built and deployed Codiquiz, an early-alpha AI-powered coding quiz platform with a React/FastAPI/PostgreSQL stack, taxonomy-driven question planning, OpenAI generation workflows, admin review tools, Batch API automation, duplicate detection, and a live preview environment.
```

## Recruiter-friendly wording

```text
Codiquiz is my portfolio product: a deployed full-stack AI platform for generating and managing coding quiz questions. It includes a public preview site, protected admin app, structured Python taxonomy, AI generation and review workflows, background workers, provider safety checks, and preview backup/operations runbooks.
```

## What to avoid saying

Avoid:

```text
Final production launch
Complete learning platform
Fully commercial product
Finished SaaS
```

Prefer:

```text
Live preview
Early alpha
Portfolio product
Production-style deployment foundation
Admin/AI-generation systems live
Learner-facing features evolving
```

## Public preview links

```text
Public preview: https://preview.codiquiz.com
Admin preview: protected demo/admin access only
```

Do not publish owner credentials.
Do not publish demo credentials unless the user intentionally wants public demo access.

## Suggested README note

```markdown
## Live Preview

Codiquiz has a live early-alpha preview at `https://preview.codiquiz.com`.

The preview demonstrates the public site, Python taxonomy browsing, protected admin workflows, AI generation planning, Blueprint coverage logic, Batch API/worker foundations, and production-style deployment practices.

This is not the final production environment. It uses isolated preview infrastructure, preview data, separate secrets, and restricted demo/admin access.
```
